# Monetization Strategy

## Purpose

Define how the product creates value and eventually becomes a sustainable business.

This document is for planning only.

It does not mean monetization will happen immediately.

---

# Core Philosophy

Revenue should come from saving users time, improving their results, and providing intelligent coaching.

Never charge for basic healthy habit tracking.

People should pay because the coach is valuable, not because features are artificially locked.

---

# Free Plan

Includes:

- Unlimited daily check-ins
- Dashboard
- Weight tracking
- Step tracking
- Workout logging
- AI macro estimation
- AI daily analysis
- Trend dashboard
- History
- User profile

Goal:

Allow users to experience the full core workflow.

# Revenue Principles

Never:

- Lock users out of logging.
- Charge for basic health tracking.
- Hide user data.

Always:

- Charge for intelligence.
- Charge for automation.
- Charge for convenience.
- Charge for personalization.

---

# Cost Awareness

The largest operating expense is AI API usage.

Design the app so deterministic calculations happen in code.

Use AI only when reasoning adds value.

Examples:

Use code for:

- averages
- streaks
- trends
- graphs
- projections
- statistics

Use AI for:

- nutrition estimation
- coaching
- recommendations
- behavior patterns
- conversations

---

# Product Flywheel

User logs more days.

↓

The coach learns more.

↓

Coaching improves.

↓

User trusts the coach.

↓

User logs even more.

↓

The coach becomes increasingly personalized.

---

# Success Metrics

Track:

- Daily Active Users
- Weekly Active Users
- 30-day retention
- Average check-ins per week
- Average AI analyses per week
- Coach Chat usage
- Weekly report views
- Subscription conversion
- Retention after 90 days

---

# Long-Term Goal

Become the easiest and smartest way to improve body composition.

Not by making users work harder.

By making healthy decisions easier.

---

# Guiding Question

Before building any feature ask:

"Would someone happily pay for this because it genuinely improves their life?"

If the answer is no, rethink the feature.
